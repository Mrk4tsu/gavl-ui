# Form

`Form` trong GAVL UI là tập hợp các class CSS để dựng trường nhập liệu rõ ràng, dễ scan và nhất quán về trạng thái. Bộ này bao phủ các thành phần quan trọng nhất của một form thực tế: `label`, `input`, `select`, `textarea`, `helper text`, `error text`, `input group`, cùng một số control như `checkbox`, `radio` và `toggle`.

Điểm cần lưu ý: GAVL UI chỉ cung cấp phần trình bày. Validation logic, submit flow, loading state, disable khi đang gửi, hiển thị lỗi từ API hay quản lý dữ liệu form vẫn do JavaScript của ứng dụng đảm nhiệm.

## Anatomy

| Class | Bắt buộc | Vai trò |
|---|---|---|
| `.gv-form-group` | Nên có | Gom `label`, field và phần mô tả thành một cụm rõ ràng |
| `.gv-form-label` | Không | Label kiểu pixel, hợp với text ngắn như `EMAIL`, `ROLE`, `PASSWORD` |
| `.gv-form-label-vi` | Không | Label dễ đọc hơn cho tiếng Việt dài hoặc câu tự nhiên |
| `.gv-input` | Có | Text input mặc định |
| `.gv-select` | Không | Select dropdown cùng hệ style với input |
| `.gv-textarea` | Không | Ô nhập nhiều dòng |
| `.gv-form-helper` | Không | Mô tả bổ sung, hướng dẫn nhập liệu, gợi ý định dạng |
| `.gv-form-error` | Không | Thông báo lỗi inline dưới field |
| `.gv-input-error` | Không | Tô trạng thái lỗi cho `input`, `select`, `textarea` |
| `.gv-input-success` | Không | Tô trạng thái thành công hoặc hợp lệ |
| `.gv-input-group` | Không | Gom field với prefix/suffix như `@`, `https://`, `.com` |
| `.gv-input-addon` | Không | Prefix/suffix đứng hai bên input |

```html
</div>
```

## Khi nào dùng class nào

- Dùng `.gv-form-label` khi label ngắn, thiên về UI token hoặc field name viết gọn.
- Dùng `.gv-form-label-vi` khi nội dung là tiếng Việt tự nhiên, dài hơn hoặc cần dễ đọc hơn pixel font.
- Dùng `.gv-form-helper` cho hướng dẫn trước khi người dùng nhập, ví dụ định dạng, giới hạn hoặc tác động của field.
- Dùng `.gv-form-error` cho lỗi inline gắn trực tiếp với field cụ thể.
- Dùng `Alert` ở đầu form khi có lỗi tổng quan, lỗi từ server hoặc thông báo submit thành công.
- Dùng `.gv-input-group` khi một phần giá trị là cố định và không nên để người dùng sửa.

## Ghi chú triển khai

- Validation state không tự sinh; bạn cần thêm hoặc gỡ `.gv-input-error` và `.gv-input-success` bằng JavaScript của ứng dụng.
- Không nên dùng `.gv-input-success` cho mọi field sau mỗi lần gõ. Chỉ hiển thị khi thực sự có giá trị xác nhận, ví dụ username hợp lệ hoặc submit thành công.
- Với form dài, nên nhóm field thành từng section nhỏ và đặt `Alert` tổng quan ở đầu section thay vì chỉ ở cuối.
- `disabled` chỉ phù hợp khi field hoàn toàn không tương tác. Nếu vẫn cần submit giá trị hoặc muốn người dùng copy nội dung, cân nhắc `readOnly`.
- Với tiếng Việt dài, ưu tiên `.gv-form-label-vi` để tránh giảm khả năng đọc của label.
